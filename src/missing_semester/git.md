# Git 从0到0.1

> 是的，0.1的理解和应用足以日常使用（对笔者而言）。

## 三步走入门Git

首先想象一个文稿。比方说，你正在写的论文。这篇论文同时有你和导师两位编辑者。这时候，就会诞生几个需求：

- 你希望可以保存**每次编辑的内容**，在必要时可以找到某个变动之后的版本（而不是搜索奇奇怪怪的文件名后缀）
- 你希望每次编辑会**留下痕迹**，例如编辑者的身份和ta的消息（导师：大改）

将文档换成一整个文件夹及其内容，上述功能就是Git的第一步：**保存变化（stage）**，**提交变化（commit）**。这个文件夹，称之为**仓库（repo）**。

然后，你还有更多需求：你希望导师可以很方便的查看和编辑你的论文

- 你希望这个仓库可以像网盘/共享文档一样直接上传到一个地方
- 别人可以复制你的仓库，并编辑成自己的版本

这个可以上传的地方，一般称为代码托管平台。Github，Gitee，都提供代码托管的功能。这是Git的第二步：**分叉（fork，复制一个别人的repo到自己的代码库）**，**克隆（clone，下载服务端的repo）**，**branch（管理分叉）**。

最后，你希望可以把导师编辑后的版本合并到你的论文中，这意味着：

- 别人可以“提出合并请求”
- 我可以查看别人的版本和我的版本的差异，并选择将哪些差异合并到我的版本

这就是Git的第三步：**提出拉取请求（pull request，提PR，请求对方拉取自己的版本）**，**合并（merge）**。

这三步涵盖了Git的基础合作流程：

1. 拉取别人的repo
2. 做出自己的编辑
3. 上传自己的版本
4. 让别人合并自己的版本

除此之外，还有一些操作，比如：

- 创建自己的repo
- 处理合并中的冲突
- 管理分支
- ...

会在后续的使用中逐个介绍。

## 边做边学

通过给项目提交变更，学习Git的基本原理与操作。

### 安装 Git

打开[Git的官方页面](https://git-scm.com/) 跟随步骤安装

### Fork本项目

> 如果没有账号，请参考[Github教学](./github.md)

打开本项目的[Github仓库](https://github.com/ethannhu/xd.hub)，点击右上角的Fork。
在新页面中填自己仓库的名字和描述，点击Create fork，Bingo！

### Clone到本地

> 如果没有ssh知识，请移步[SSH教学](./ssh.md)，并参考[Github教学](./github.md)配置SSH密钥

打开自己的Fork到的repo，点击绿色的Code，选择Clone-Local-SSH，文本框中的地址应该是 `git@github.com:xxx/yyy.git`，xxx是你的账户名，yyy是你的repo名。复制这个地址。
打开自己的终端（windows上则是git bash），输入git clone 这个地址 。

如果没有出现错误，终端当前工作目录下就会出现你希望拉取的项目，Bingo！

### 做出自己的编辑

> 这里以VS Code为例子，JetBrains的产品应当也有类似功能，例如PyCharm和IDEA。

用VS Code打开本地的项目。做出一些编辑，比如修改一些Markdown文件？然后保存。
这时候，左边栏（如果你的VSC是默认布局）里面的Source Control图表会出现一些数字。

点击之后，你的左侧页自上到下会显示Changes和Graph。点击Changes里面的文件，你应该可以看到文件的变化（Working Tree）。对于每个文件，会有Open，Discard和Stage三个按钮。

对于你想保存的变化，点击Stage，文件会移到Staged Changes中。当你挑选好需要保存的变化，在Message栏中填写你想留下的消息（比如说：“修复了xxx中的笔误”），然后点击Commit。

这时候，Commit按钮会变成Sync。你可以点击Sync，这时你在本地Stage过的变更将带着Commit Message一起上传到远程的repo。

至此，你已经完成了Git之旅的伟大前半场！现在你可以随便fork你喜欢的项目，或创建自己的项目，并把他们整齐保存在Github中。
