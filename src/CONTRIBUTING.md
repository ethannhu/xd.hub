# 贡献指南

感谢关注 **xd.hub**！非常欢迎你分享资源，帮助更多同学。

## 参与方式

你可以通过以下几种方式参与共建：
1. **完善已有内容**：修正错误、补充缺失的实验步骤或更新课程资料。
2. **新增资源**：上传新的实验攻略、大作业代码或经验分享。
3. **提交 Issue**：反馈错误、建议新功能或请求特定的课程资料。

---

## 贡献守则

在提交资源前，请务必注意：

1.  **脱敏处理**：请务必删除文档或代码中的个人隐私信息（如**姓名、学号、班级**等）。
2.  **版权与合规**：
    - 确保分享的内容不违反学校的**保密**规定。
    - 严禁上传含有**版权**争议的付费电子书、**未公开**的考试试卷原件。
    - 引用他人成果（如论文、博客）请注明来源。
3.  **禁止垃圾内容**：请勿上传与课程学习无关的推广信息。

---

## 组织与命名规范

为方便维护，请遵循以下规范：

- **目录结构**：按照 `学院 / 课程名 / 资源名` 的层级进行存放。
    - 上传单文件时应上传到对应的课程资源文件夹中
    - 上传项目时应包含在根目录包含README.md，在其中介绍项目内容
- **命名建议**：
    - 使用清晰的snake_case英文命名，避免使用 `111.pdf` 或 `新建文档.docx`。
    - 示例：计算机网络实验一：`computer_networking_experiment1.md`。
- **文件格式**：
    - 文本类优先使用 **Markdown (`.md`)**，方便版本追踪和在线阅读。
    - 代码类请移除不必要的中间编译文件（如 `.exe`, `__pycache__`）。

## 提交贡献

### 使用 Pull Request

1. **Fork** 本仓库到你的账号。
2. **Clone** 你的远程仓库到本地。
3. **Create** 创建一个新的分支：`git checkout -b contrib/your-feature-name`。
4. **Commit** 提交你的更改，并撰写简明扼要的提交信息。
5. **Push** 推送到你的 GitHub：`git push origin contrib/your-feature-name`。
6. **Open PR** 在本仓库发起 Pull Request。
> 如果对于Git的使用方法存在疑惑，请移步[Git](./missing_semester/git.md)。
